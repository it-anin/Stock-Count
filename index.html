1. R01.102 — จำนวนสินค้า (อัพได้หลัง 21:00)
  • รายงาน R01.102 → สร้างรายงาน → เลือกสาขา → ติ๊กสินค้าทั้งหมด → ส่งออก .CSV
  • การอัพโหลดจะ reset ข้อมูลการนับทั้งหมดให้เริ่มใหม่
2. R16.104 — ยอดขาย/รับเข้าระหว่างนับ (อัพก่อน Confirm)
  • รายงาน R16.104 → สร้างรายงานตามวันที่ → ประจำวัน → ส่งออก .CSV
  • ระบบนำ ORCM/OCTM (ยอดขาย) บวกกลับ และ OTFB/ORTS/OTFI (รับเข้า) หักออก
  • ใช้คอลัม R (BASEQUANTITY) — แปลงเป็นหน่วยเล็กสุดแล้ว

ขั้นตอนการนับสต็อก:

1 เลือกสาขา → ใส่ PIN → เลือกชื่อผู้ใช้
2 หลัง 21:00 → อัพโหลด R01.102 (reset ข้อมูลนับ + เริ่มวันใหม่)
3 สแกน Barcode หรือ SKU → กด Enter — จำนวนสะสมอัตโนมัติ
4 อัพโหลด R16.104 ก่อนกด Confirm (ยอดขาย/รับเข้าระหว่างนับ)
5 กด ✔ Confirm → ระบบเปรียบเทียบกับ System Qty อัตโนมัติ
6 รายการ ⚠️ Audit → เภสัชเปิด Audit Verify → สแกนนับซ้ำ → กด ยืนยันทั้งหมด
7 ครบ 100% → เริ่มรอบใหม่ อัตโนมัติ
⚠️ กฎพิเศษ
  • สแกน SKU เดิมห่างกันเกิน 2 นาที → จำนวนถูก reset เป็น 0 ใหม่
  • คอลัม Qty ซ่อน ถ้าจำนวน ≤ 100 (ป้องกัน bias) — แสดงเมื่อ > 100
  • สินค้า Pass / Audit / Stock Adj แล้ว → สแกนซ้ำไม่เปลี่ยนจำนวน

สถานะผลลัพธ์:

✅ Passจำนวนตรงกับระบบ
⚠️ Auditจำนวนไม่ตรง — รอเภสัชตรวจซ้ำใน Audit Verify
🔴 Stock Adjตรวจซ้ำแล้วไม่ตรง — ต้องปรับสต็อก
❓ Unknownไม่พบ Barcode ในระบบ
